# Mini Search (Node.js + Jest + GitLab CI)

Мини-проект: индексирует строки из файла и выполняет поиск по подстроке.
Используется для демонстрации автотестов и CI-пайплайна.

---

## Установка проекта (локально)

Открываем терминал в папке проекта и выполняем:

npm install

---

## Запуск тестов локально

npm test

После запуска:

* в консоли появится результат тестов (PASS/FAIL)
* создастся папка **coverage/** — отчёт покрытия
* создастся **reports/junit.xml** — JUnit XML отчёт

---

## Проверка падения тестов

Можно специально сломать любой тест, например:

expect(2+2).toBe(5)

После этого:

npm test

Тесты упадут — это показывает, что CI тоже будет падать при ошибках.

---

## Что делает CI (GitLab CI)

Pipeline автоматически запускается при:

* push в репозиторий
* merge request

CI выполняет:

1. Поднимает Node.js окружение
2. Устанавливает зависимости (`npm ci`)
3. Запускает тесты (`npm test`)
4. Генерирует:

   * coverage отчёт
   * JUnit XML отчёт
5. Сохраняет их как **artifacts**
6. Пайплайн падает, если хотя бы один тест не прошёл

---

## Структура проекта

src/ — исходный код
tests/ — тесты Jest
corpus.txt — файл для интеграционного теста
.gitlab-ci.yml — конфигурация CI

---

## Требования задания

* ≥6 unit тестов 
* ≥2 интеграционных теста 
* ≥1 негативный тест 
* JUnit XML отчёт 
* Coverage отчёт 
* CI stage test
